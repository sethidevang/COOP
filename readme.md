## Description

This project presents a Systematic Literature Review (SLR) focusing on the challenges in adopting microservices architecture. The study analyzes key aspects such as:

* Data Consistency
* Fault Tolerance
* Scalability Trade-offs

The research is based on papers collected from major academic sources including IEEE Xplore, ACM Digital Library, SpringerLink, ScienceDirect, and Google Scholar (2016–2026).

---

## Key Findings

* Strong consistency models limit scalability
* Eventual consistency is widely adopted in microservices
* Saga pattern is preferred over Two-Phase Commit
* Microservices increase system complexity and failure risks
* Lack of standardized benchmarking frameworks
